**User :** Bug trouvé ! La saisie tapée dans le textarea n'est pas utilisée par l'IA. Exemple : "Crée une image en basse résolution (256x256) d'une scène intérieure d'une maison contemporaine" → résultat "Peindre une image détaillée d'un paysage nocturne avec des étoiles...". Analyse du user : "le prompt envoyé à l'IA n'inclut pas ton texte".

**Assistant :** Reproduction du bug confirmée via curl. Le problème est double :

### Bug 1 : `system_instruction` manquant
Le frontend mettait le system prompt et le message utilisateur dans les mêmes `parts` de `contents`. Gemini les traite comme un seul bloc et n'utilise pas correctement la consigne.

**Fix (frontend) :** Envoyer `system` séparément du `contents` quand un proxy est utilisé :
```js
body: proxyUrl
  ? { system, contents: [{ role: 'user', parts: userParts }] }
  : { contents: [{ role: 'user', parts: [{ text: system }, ...userParts] }] }
```

**Fix (proxy) :** Accepter `body.system` et l'utiliser comme `system_instruction` dans la requête Gemini :
```js
if (system) geminiBody.system_instruction = { parts: [{ text: system }] };
```

### Bug 2 : Fallback Groq trop faible
Le modèle `llama-3.1-8b-instant` ne suit pas l'instruction JSON. Résultat : réponse en texte libre au lieu du format JSON attendu.

**Fix :** Passer à `llama-3.3-70b-versatile` + `json_object` mode :
```js
body: JSON.stringify({model:'llama-3.3-70b-versatile', messages, max_tokens:2048, response_format:{type:'json_object'}})
```

- `inline_data` → `inlineData`, `mime_type` → `mimeType` (Gemini attend du camelCase)
